Meeting I/O for AI agents.

samoagent is not a full-fledged agent. It is the helper process you give to an AI agent so it can join a call, watch the transcript, send chat, and inspect the current screen.

View source npm package
agent session 
npm install -g samoagent

export RECALL_API_KEY=...

samoagent join "https://meet.google.com/..." \
  --ws-video \
  --name Leo \
  --dict postgresfm

samoagent watch
samoagent frame
samoagent chat "I can see the screen now."
samoagent leave

Tools for call participation

Give these commands to an AI agent and it gets a controlled interface to the live meeting.

Join

Starts a Recall.ai bot for Zoom or Google Meet and keeps local state for the active session.

Watch

Streams one utterance per line so an agent can follow the meeting in real time.

Frame

Receives WebSocket video frames in memory and writes files only when the agent asks.

Chat

Sends deliberate meeting chat messages when the agent needs to participate visibly.

Leave

Removes the bot, stops local processes, and cleans up runtime state.

How it integrates

  • Your AI agent runs samoagent commands as tools: join, watch, chat, frame, and leave.
  • samoagent starts a local callback server and a Recall.ai bot for the target Zoom or Google Meet call.
  • Recall.ai sends transcripts, chat delivery, and WebSocket video-frame events back to the local server.
  • The normal local path uses ngrok http for those HTTPS/WSS callbacks. Free ngrok should be enough for personal --ws-video use.
  • ngrok TCP is only for the optional RTMP mode and may require card verification.
  • Webhook and frame routes are token-protected, and default runtime files stay under ~/.samoagent/.
